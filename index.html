Getting Started

Medi.co is a design system created by the Medico in order to help teams build quality ehealth and digital experiences accessible across all platforms.

If you are looking for resources on style guides, check out material.io. This style guide is an add on to the current Materials UX style guide hosted by google, it is adapted to better support and amplify voices in ehealth through malleable and intentional design.


You can view a case study of Medico here and you can view the additional medico assets on figma here.

Colors

The Medico Design color system helps you apply color to your UI in meaningful way. In this system, colors are assigned to the main systems in the body. For additional prognosis colors can be represented as dots or lines, but never as solid shaded cards. Individual colors are collated next to the charting notes in question that can be linked to one or more corresponding colors.

Theming

For the Medico system the baseline system should be a neutral color like cream or offwhite. Textures for Medico are not recommended. Where possible be sure to implement a dark and light version of the theme for ease of use and reducing eye strain.

Buttons

Usage

Buttons communicate actions that users can take. They are typically placed throughout your UI in places that require user interactions. Sometimes buttons are part of the keyboard interface and do not need to be directly displayed.


Principles

Identifiable

Buttons should indicate that they can trigger an action.

Findable

Buttons are easily found and are accessible at all times.

Types of Buttons

There are 3 types of buttons that have modified uses in Medico

Toggle Buttons

Toggle buttons group a set of actions using layout and spacing. Toggling implies they can be turned on or off through a singular action. Toggle buttons are rarely used in Medico.

Do
Do Not

Floating Action Buttons

Floating action buttons are buttons that use hierarchy to denote emphasis.Floating Action Buttons or FABS are usually always present on whatever tab screen they are on. This ensures that medical professionals are able to quickly add information to the patient's medical history.

Do
Do Not

Contained Buttons

Contained buttons group actions that are similar together in the text field, and are used to quickly categorize information by breaking up a workflow into smaller easier to handle steps.

List Cards

Usage

List cards communicate information that is essential for medical technicians. They are the cards that medico users interact the most with.

Principle

Identifiable

List Cards should be identifiable as a clear call to action.

Interactable

List Cards should have an interaction component to be able to be adjusted for medical professionals.

Clear

List Cards and their purpose should be clear and easy to understand.

List Card Selector and Modifiers

Swiping

As a part of the initiative to be clear and concise, List Cards come with an added quality of life feature for medical professionals. Swiping to the Left will reveal a hidden panel for additions after a card has been created, medical professionals have the options to add pictures, or highlight certain messages.


Doctor Note

Attending doctors have the added option to add notes to List Cards when fully deployed, these notes can help clarify measures taken from the notes already created.


Picture Function

List Cards have an added picture function to help document progressing or healing illnesses.


Alert

An Alert function has been added to List Cards to notify attending doctors of any warning signs. These list cards are highlighted in red.

Tabs

Usage

Tab organizes content across different screens, data sets and other interactions. Tabs organize and allow navigation between groups of content that are related and at the same level of hierarchy.


Tabs in a set

Each tab should contain content that is distinct from the other tabs in a set. For example, tabs can present different surgery rotations or patient check ups, or even different variants of health care check ups.


2+1 Schema Methodology

Tabs in Medico function differently from Google Material. Tabs in Medico use a 2+1 schema to always reserve a tab for an emergency EMS call. This tab should always be highlighted in red and positioned on the right top corner.

Text Input

Usage

The textbox is used to fill out forms and list cards with information.

Principles

Discoverable

Text fields should stand out and indicate users can input information.

Moddable

Text fields should be modular and have multiple access points in which the user can highlight and select additional information for the list cards to hold.

Efficient

Text fields are an extension of the list card, and should make it easy to understand the information that is requested and placed.

Assistive Elements


Selectors

List cards have been updated to feature selectors to help medical professionals quickly fill out forms. Modifications to the text in the Listcard can be done by selecting a modifier that either modifies the type of List Card or the Body System.


Selector for Body Systems

Selectors for body systems use the color/symbol system outlined in the colors tab. By allowing Medical Professionals to quickly inform doctors of the body systems responsible for the ailment. Body System selectors are only used for the onboarding of a new illness or symptom in the patient's history section.

Do Not

Selectors for List Card

Selector for List cards use a different system than the color/symbol system. This new system for selectors allows medical professionals to add notes, or flag notes for more critical inspection. Senior officers can also add doctors addendums recommending additional treatments through analysis of the patient's history section.
